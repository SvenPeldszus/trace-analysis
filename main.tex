\begin{abstract}
	Trace links are important building block in security engineering.
	In software projects we have different kinds of trace links, external trace links that connect different artifacts, and internal trace links within an artifact that express dependencies between elements of the system.
	These trace links support the security engineering by providing a rationale to developers and can even be leveraged to automatically perform security tasks, such as prioritizing issues according to their security impact on the project.
	However, it is unclear what are the characteristics of good trace links, i.e., which ones carry the most security-related information.
\end{abstract}
